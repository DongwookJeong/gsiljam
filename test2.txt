git 배워야되는 사항들
1. merge 나 pull 하기전에 대상 파일을 확인하는 방법. 확인하고 merge해야 하기 때문
2. branch checkout 다루는 법

오늘 질문할 사항들
박재형 :{
  1. api제작법
  2. xml 다루는 법
  3. 보안화 암호화 작업
  4. 프로젝트가 합당한가.
}
정동욱 :{
  1. api 생성이 필수인가? 불러와서 진행하려고 하는데 상관이 없는가?
  2. 프로젝트 진행에 구체적인 구조를 잡고 가는 것이 좋은가?
  - 아니라면 진행 도중에 맞춰가는 것이 좋은가?
  3. 모듈화를 함수별로 진행 할 경우 세분화하여 처리하는 것이 좋은가?
  4. 마감 시간을 정해서 분배를 하는 것이 좋은가?
}

정정원 : {
  1. 어떠한 프로젝트를 실행하기 전에 전체적인 큰 틀을 어떤 기준으로 잡고 시작하는 지
  2. 구체적으로 작업의 시간안배를 어떤 형식으로 잡고 진행을 하는지
  3. 소규모 프로젝트를 진행할 경우와 조금 규모가 있는 프로젝트를 진행할 경우의 차이나 진행방식이 서로 다른가
  4. 개발자들이 생각하는 프로젝트에서 가장 중요하게 생각해야 하는 부분
}
